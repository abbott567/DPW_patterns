Add items to a list

This pattern is useful in the design of forms when you require the user to complete a list of things related to the service, for example medical conditions or names of people. The concept is that the user is not presented with too many empty fields to begin with and instead adds only as many as they need.

Standard version - horizontal

This is how the first list item would appear. Note the grey colour of the button below.

Once the user clicks the 'add another' button a second set of text entry fields appears below, separated by a keyline. The 'add another' button moves below the new fields ready for the user to click it again.

Note that a 'remove' link also appears next to both of the items and this allows the user to quickly delete an entry if they need to. If there are only two items in the list, the 'remove' button will disappear once it is clicked the first time because the user should not be able to remove all items.

Demo version

Use the below 'live' version to see how it functions. We've limited the number of items you can add to 5 for the purposes of the demo.

Variant version - vertical fields

Sometimes you may need to ask for more fields than will fit horizontally and so there's this variant with vertical fields which works in exactly the same way.
